# Promoting PrivaQuest

This niche is smaller than "URL shortener" but the **audience has real pain**. Target people who already worry about GDPR — not generic dev Twitter.

## Before posting

- [ ] Record GIF: submit form (DE) → admin dashboard → audit log
- [ ] GitHub About filled (see docs/market-research.md)
- [ ] Topics: `gdpr`, `dsgvo`, `rgpd`, `self-hosted`, `compliance`
- [ ] Pin repo
- [ ] Tag v0.1.0 release

## Where (by market)

### English
- **Show HN**: "Show HN: Self-hosted GDPR request queue for small teams"
- **r/selfhosted**, **r/gdpr**, **r/golang**
- **Dev.to**: "What happens when someone emails 'delete my data'"

### Germany
- **Habr / Heise** (technical, not salesy): "DSGVO-Anfragen operationalisieren"
- LinkedIn posts in German targeting KMU IT
- Hetzner forum / German privacy communities

### France
- Dev communities + "RGPD PME" angle
- Mention CNIL context, OVH hosting — don't claim legal certification

## Post template (EN)

> Small EU teams get GDPR emails in the inbox and panic. Built a self-hosted queue: intake form, 30-day deadline, audit log. EN/DE/FR. Runs on your server — no US processor. Feedback welcome from anyone handling DSARs manually.

## Post template (DE)

> DSGVO-Anfrage per E-Mail, niemand trackt die Frist? Self-hosted Tool: Formular, 30-Tage-Deadline, Audit-Log. DE/EN/FR. Läuft auf eigenem Server.

## What NOT to say

- "GDPR certified" — you're not
- "Replaces your lawyer" — instant distrust
- "Enterprise-grade" — AI smell

## Metrics that matter

- Clones from DE/FR (GitHub Insights → Traffic)
- Issues from people actually running it
- Inbound from DPOs / agency owners
